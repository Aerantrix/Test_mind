❌ Не работает (функциональные баги)

Таймкод Что произошло Серьёзность
~07:00 После того, как доступ к документу был отозван (revoked), при попытке открыть его пользователь получает сырой XML-вывод браузера с сообщением «this XML file does not appear to have any style information… document tree is shown below… code: no such key, the specified key does not exist». То есть вместо нормальной странички «У вас нет доступа / документ удалён» — техническая XML-ошибка S3. Серьёзный UX-промах + потенциально утечка деталей backend (key, structure). 🔴 High
~05:30 После отправки чего-то в чат: «On my end, it’s just a blank message». Отправитель видит пустое сообщение в своей ленте. На другой стороне видно нормально. 🟠 Medium
~02:00 В чате с переводом есть переключатель «transcriptions: expanded / collapsed / hidden» — но он не работает: «when I change the transcriptions to expanded or collapsed or hidden, it doesn’t seem to change anything». 🟠 Medium

✅ Что работает хорошо

  • Edit history сообщений в чате: «I see the edit updates» — после редактирования видна история.
  • Toggle original / translated для сообщения работает: «toggling the Russian part on and off».
  • Загрузка файла-документа в чат: «Text file, I load it, it comes up with the instructions. Seems to be working».
  • Открытие документа по прямой ссылке в новой вкладке — пока есть доступ — работает: «I can still see the document when I post the link into a separate tab».

🚦 Приоритеты

  1. 🔴 Сырая XML-ошибка вместо «нет доступа» (~07:00). Двойной риск:
    • UX: пользователь не понимает, что произошло.
    • Security: страница раскрывает backend-детали (S3 key, ошибка NoSuchKey).
  2. 🟠 Сообщение видно как blank у отправителя (~05:30). Похоже на проблему с rendering attachment / link на стороне sender.
  3. 🟠 Toggle expanded/collapsed/hidden у transcriptions (~02:00) — функция не работает, нужно либо чинить, либо убирать.
