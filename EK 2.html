~08:00–10:00 EK искал «change password» в Settings → не нашёл. Полез в Help → AI-ассистент объяснил, что паролей нет, логин по одноразовому коду. UI настроек сам этого не объясняет — там пусто. Пользователь логично ищет «изменить пароль», не находит, тратит время.

~10:30 После delete + повторного login на тот же email: EK не уверен — это новый аккаунт или восстановление старого. Пришлось делать второй прогон с пометками, чтобы убедиться, что аккаунт действительно был удалён, а не «приостановлен».

✅ Что работает хорошо

Регистрация через email + one-time code

  • Email-код приходит «very quick».
  • «My code with your intermind sign-in code expires in 10 minutes or click the button to sign-in directly» — формулировка письма понятная.
  • Кнопка «sign-in directly» работает — не надо вводить код руками.

SSO

  • Google login: «worked perfectly fine on both website and phone» (использовал раньше).
  • На UI явно показаны Google / Microsoft / SSO как опции.

Help / AI assistant

  • Когда EK не понял, что паролей нет, AI-ассистент в Help ответил быстро и ясно: «Intermind does not use passwords for email logins. You receive a one-time six-digit code…». Это спасло сценарий.

Delete account — образцовый flow ⭐

EK подробно хвалит каждый шаг: - «Very clear red text», использование иконки корзины — «universal symbol, very easy to understand». - Чёткое заявление о необратимости: «very clear statement that it is permanent». - Двойное подтверждение — второй pop-up с предупреждением (важно против мискликов). - Валидация email при подтверждении: при неверном email — «email does not match» (EK проверил специально). - Большой красный warning перед финальным delete: «This will permanently delete your account and all data including meetings, messages, files, team ownership and subscription. This cannot be undone». - Реальная проверка после удаления: EK создал данные → удалил → залогинился обратно тем же email → «the stuff that I added before isn’t there. So that does seem to be working». Удаление действительно сносит данные.

Логин повторно тем же email после delete

  • Аккаунт успешно пересоздаётся одним и тем же email — это правильное поведение.
  • Что было до удаления — не возвращается (правильно).

📋 Что не дотестировано

  • Восстановление пароля через «Forgot password» — пункт сценария §2; EK его не делал, потому что паролей нет. Можно вычеркнуть пункт из инструкции.
  • Истёкшая / невалидная ссылка на странице /confirm — частично проверено (кликнул на ссылку из письма), но повторного перехода (на «использованную ссылку») и сценария «открыть через час» не было.
  • Регистрация через Microsoft SSO — не пробовал (использовал Google и email).
  • Невозможность зарегистрироваться с уже использованным email — частично затронуто, но без явного «попытался заново и получил ошибку».

🚦 Приоритеты

  1. 🟠 Auto-detect языка в Edge ведёт на DE-локаль (00:00). Чинить либо логику определения, либо показывать баннер «We detected DE — switch to EN?» вместо принудительного редиректа.
  2. 🟠 Settings → нет блока «Sign-in method» (~08:00). Добавить явный текст «You sign in with one-time codes — passwords are not used» прямо в настройках, не прятать в Help.
  3. 🟡 Главная для залогиненного — нет очевидной точки входа (~04:30). Когда пользователь уже вошёл, главная не должна выглядеть как landing.
  4. 🟡 Update инструкции для тестировщика: вычеркнуть «восстановление пароля» из §2 сценариев, поскольку паролей в продукте нет — заменить на «повторный one-time login».

🎉 Что особенно хорошо

Delete account — лучшая часть Inter-Mind по словам EK. Это редкость: обычно delete-flow проектируют небрежно. Здесь он выглядит чётко проработанным — двойное подтверждение, валидация, явные предупреждения, реальная очистка данных.
