❌ Не работает / отсутствует

Таймкод Что произошло Серьёзность
~05:00 Документ доступен по прямой ссылке без авторизации: «It opens the link, the whole document is opened on a separate page… I’ll log out from my account on another browser, try to open the same link, looks like I still have access to this document». Любой, у кого есть ссылка, может открыть документ, не залогинившись. Это security/privacy баг (если документы предполагались как «только для команды»). 🔴 High (security)
~04:30 При попытке отправить ссылку на документ в чат — сообщение становится blank у обеих сторон: «I’m not sure if that’s shown up on your end. On my end it’s just a blank message. Yeah, I also see the empty message from you». Подтверждение находки из EK 11-16, теперь видно, что баг на обеих сторонах, не только у sender. 🟠 Medium
~02:30 Domain Management — недоступно: «Create your own corporate domain — there is no feature for that right now». В UI Integrations этой опции нет. По доке должна быть для paid plans. 🟠 Medium (gap)
~02:45 Google Drive integration — недоступно: «Connect Google Drive to your account through OOS — there is no feature for that right now». 🟠 Medium (gap)
~07:00 AI editor completion — функция отсутствует / не находится: «AI-supporting editor. У нас нет редактор-файлов». 🟠 Medium
~06:00 Toggle expanded/collapsed/hidden у transcription не работает (подтверждение EK 11-16): «It doesn’t seem to change anything». 🟠 Medium
~06:30 Standalone Translate page (/translate) не нашли: «Open Translate as a separate tool. Type any text, I think it’s a chat feature». Oscar не понял, что это standalone, искал в чате. 🟡 Low
~12:00 После обработки документа меняется визуализация ошибки: «this XML file does not appear to have any style information… code: no such key, the specified key does not exist» — снова XML-ошибка S3 (как у EK 11-16). 🟠 Medium (UX + security info leak)

✅ Что работает

Интеграции

  • Calendar (Google) подтверждён — через Scheduled Meeting → Add to calendar (а не из страницы Integrations).
  • Backup & Restore — упомянут как что-то существующее в Integrations: «и Backup and Restore». Не понятно, что именно делает.

Документы

  • Document upload в чат работает.
  • После delete документ становится недоступен по ссылке (см. ошибку выше, но факт revoke срабатывает).

Mobile

  • Join meeting по ссылке с телефона работает без полного логина: «So I joined the meeting without signing in on my phone. This part works well».

📋 Что не покрыто (большая часть §13–§15)

§13 Безопасность

  • SSO / SAML — нужен Business/Enterprise аккаунт, нет доступа: «нужно ехзернный аксесс от менеджера».
  • Privacy Zones — не проверено.

§14 Поддержка

  • Live chat support — не проверили: «yeah, I think it also requires other account with enterprise and business tariff». Спорно — по сценарию live chat на всех тарифах, не только Business.

§15 Мобильный

  • Открытие митинга по ссылке — да.
  • Перевод на мобильном — нет.
  • iOS Safari отдельно, Android Chrome отдельно — нет.
  • Wi-Fi → 4G переключение — нет.
  • Фоновый режим — нет.

🚦 Приоритеты

  1. 🔴 Документы доступны по прямой ссылке без логина — серьёзный security/privacy баг. Если документы внутрикомандные — это утечка. Сейчас же — ссылка работает в режиме «public, anyone with link».
  2. 🟠 Domain Management отсутствует в Integrations — gap между обещанным и реальным.
  3. 🟠 Google Drive / OneDrive отсутствуют — то же самое.
  4. 🟠 AI editor completion не находится / отсутствует.
  5. 🟠 Пустое сообщение при copy/paste ссылки — рендеринг сломан с обеих сторон.
  6. 🟠 XML-ошибка при revoke (подтверждение EK 11-16).
  7. 🟡 Live chat support — уточнить тариф: оно для всех или только для Business+? Сейчас тестировщики не уверены.
  8. 🟡 Standalone Translate (/translate) — добавить явный вход в UI, чтобы не путали с chat feature.
