Oscar step 7-8 — Inter-Mind UX-отчёт

UX-отчёт: Oscar — Step 7 (Чат в встрече) + Step 8 (Постоянный чат + Note editor)

Тестировщик Oscar + EK
Разделы §7 Чат в встрече, §8 Постоянный командный чат + Note editor
Длительность ~16:02
Видео Oscar step 7-8.mp4

❌ Не работает / странное

Таймкод Что произошло Серьёзность
~10:00 Лимит «message too long for translate, maximum is 1000 characters» при копировании большого текста из Wikipedia в чат. Это per-message ограничение, не явная характеристика тарифа. На pricing-странице обещаны лимиты в «10K/200K/3M words» — это другое (агрегированный лимит на тариф). Пользователь не понимает связи и видит «1000 chars max» как баг. Сообщение приходит в оригинале, без перевода. 🟠 Medium
~12:00 «Есть небольшие баги в чате при виде перевода. В некоторых местах он на русском, в некотором на английском» — UI не консистентен: одна и та же лента показывает разные сообщения в разных языках без явного контроля. 🟠 Medium

✅ Что работает

Чат внутри встречи

  • Двусторонний перевод сообщений работает: «первое сообщение split в два, где есть русский справа и автоматически переводится в английском слева».
  • «Все работает хорошо в этой части» (общая оценка перевода в чате).

Чат после встречи / постоянный чат

  • История чата сохраняется: «чат, который открывается во время созвона, такой же, как и во время нашего чата вне созвона, продолжился».

Создание канала после добавления контактов

  • Подтверждение нюанса из §3: канал без контактов не создаётся, но из встречи можно добавить участника в контакты через клик по профилю.
  • После добавления: «теперь у меня есть канал с Оскаром».

Note Editor ⭐

  • Создание заметки из чата работает.
  • Заметка видна у второй стороны: «Try to find a note of some kind, if you see it in the chat… Да, вот она».
  • Удаление заметки распространяется на обе стороны: «delete the note… It has disappeared».
  • Перевод заметки на язык получателя работает.

❓ Неудобно (продолжение)

Таймкод Что отметил
~07:00 Чтобы создать канал, нужны контакты. Контакты добавляются только из meeting (через профиль). Это «немного некомфортно» — нужен workaround «зайди в meeting, чтобы добавить контакт». Должен быть прямой путь «добавить контакт по email» вне встречи.

📋 Что не дотестировано

  • Прикрепление файлов в чат (упоминается мельком, но не показано как тест).
  • Эмодзи в чате.
  • Пересылка сообщений / copy link / mention.
  • Поиск по сообщениям.
  • Создание встречи из чата (новая фича §8) — Oscar упомянул «button for join», но не уточнил.
  • Уведомления о новом сообщении вне фокуса.

🚦 Приоритеты

  1. 🟠 Лимит 1000 символов на сообщение + расхождение с обещанием pricing (10K/200K/3M words): объяснить пользователю, что эти числа значат, и почему сейчас стоп на 1000 chars.
  2. 🟠 Inconsistency перевода в ленте чата — одна лента, разные языки в разных пузырьках.
  3. 🟡 Add contact вне meeting — добавить прямой путь.
  4. ✅ Note editor — образцово работает end-to-end.
