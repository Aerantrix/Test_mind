# Inter-Mind — UX-тестирование (alpha)

Результаты ручного тестирования продукта Inter-Mind. Сводный отчёт + 16 детальных отчётов по сессиям + транскрипты речи тестировщиков.

**📊 Открыть сводный отчёт:** [`index.html`](./index.html)

## Содержимое

| Файл | Что |
|---|---|
| `index.html` | Сводный одностраничный отчёт — точка входа для разработки |
| `<session>.html` (×16) | Детальные отчёты по каждой сессии тестирования |
| `style.css` | Общий стиль |
| `transcripts/<session>/transcript.txt` | Полный транскрипт речи тестировщика |
| `transcripts/<session>/transcript.srt` | То же с таймкодами (можно открыть в плеере как субтитры к видео) |
| `transcripts/<session>/transcript.vtt` | WebVTT-формат для веб-плееров |

Сессии:

- **Oscar** — 8 сессий, шаги 0–15 сценариев
- **EK** — 8 сессий, те же шаги параллельно

Cross-tester verification: большинство критических багов воспроизведены обоими тестировщиками независимо.

## Метаданные

- **Дата сессий:** 2026-06-08
- **Длительность:** ~3.5 ч аудио (16 видео)
- **Тестировщики:** Oscar, EK
- **Метод записи:** OBS Studio (видео + раздельные аудио-дорожки)
- **Обработка:** Whisper large-v3-turbo + Silero VAD (для устранения hallucination-loops)

## Как читать отчёт

1. Открой [`index.html`](./index.html) — увидишь все находки, разнесённые по приоритетам:
   - 🔴 Топ-баги (критические)
   - 🟠 Топ-важных (medium)
   - 🟡 Косметика и UX-трение
   - ✅ Что работает отлично
   - 📋 Что не покрыто
2. В колонке **«📂 Детали»** — кликабельные ссылки на отчёт конкретной сессии, где найден баг (с таймкодом и цитатой тестировщика).
3. В детальном отчёте сверху — блок **«📦 Материалы сессии»** со ссылками:
   - 📄 Транскрипт (плоский текст)
   - 📝 С таймкодами (SRT — можно прицепить к видео в плеере)
   - 📹 Видео (см. ниже)

## Видео

Видео-файлы (~3.7 ГБ суммарно) **не загружены в репо** — git не предназначен для бинарных файлов такого размера. Они лежат отдельно.

В каждом детальном отчёте есть плейсхолдер «📹 Видео — добавить ссылку Google Drive». Нужно подменить `href="#"` на реальную shared-ссылку (Google Drive / YouTube unlisted / любой видеохостинг).

**Шаблон обновления для одной сессии:**

```bash
# Найти и заменить
sed -i '' 's|href="#" class="placeholder" data-needs-url="1"|href="https://drive.google.com/file/d/XXX/view"|' "Oscar Step 0-1.html"
```

Или массово, если все ссылки в одном Google Drive folder — собрать через скрипт.

## Деплой на Vercel

Это статический сайт — никакой сборки не нужно.

1. В Vercel: **New Project → Import from GitHub → выбрать `Aerantrix/Test_mind`**.
2. **Framework Preset:** Other (или Static).
3. **Build Command:** оставить пустым.
4. **Output Directory:** оставить `.` (корень репо).
5. Deploy.

Получится `https://test-mind-xxx.vercel.app/` → откроется `index.html`.
