❌ Doesn't work (functional bugs)

Timecode What happened Severity
~07:00 After access to a document was revoked, when trying to open it the user gets a raw XML browser output with the message "this XML file does not appear to have any style information… document tree is shown below… code: no such key, the specified key does not exist". That is, instead of a normal "You don't have access / document deleted" page — a technical XML S3 error. Serious UX miss + potentially a leak of backend details (key, structure). 🔴 High
~05:30 After sending something in chat: "On my end, it's just a blank message". The sender sees a blank message in their feed. On the other side it appears normally. 🟠 Medium
~02:00 In the chat with translation there is a "transcriptions: expanded / collapsed / hidden" toggle — but it doesn't work: "when I change the transcriptions to expanded or collapsed or hidden, it doesn't seem to change anything". 🟠 Medium

✅ What works well

  • Edit history of chat messages: "I see the edit updates" — after editing, the history is visible.
  • Toggle original / translated for a message works: "toggling the Russian part on and off".
  • Uploading a document file to chat: "Text file, I load it, it comes up with the instructions. Seems to be working".
  • Opening a document via a direct link in a new tab — while access exists — works: "I can still see the document when I post the link into a separate tab".

📋 Not fully tested

  • Effectively §11 (Integrations — Calendar, Google Drive, OneDrive, Domain Management) not completed.
  • §12 (Documents with permissions — partially touched on via revoke; AI editor completion — not mentioned; Translate page standalone — none).
  • §13 (Security) — none.
  • §14 (Support) — none.
  • §15 (Mobile) — not in this session.

🚦 Priorities

  1. 🔴 Raw XML error instead of "no access" (~07:00). Double risk:
    • UX: user doesn't understand what happened.
    • Security: page exposes backend details (S3 key, error NoSuchKey).
  2. 🟠 Message appears as blank for the sender (~05:30). Looks like a problem with rendering the attachment / link on the sender side.
  3. 🟠 Toggle expanded/collapsed/hidden for transcriptions (~02:00) — feature doesn't work, need to either fix or remove.
