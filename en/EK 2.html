❌ Doesn't work (functional bugs)

Timecode What happened Severity
00:00–02:30 Visiting intermind.com from Microsoft Edge automatically opens the German version of the site, even though the user is English-speaking. Reproduced twice in a row. EK suspected a VPN, checked — none. Resolved manually via the language switcher in the header. → locale auto-detect in Edge is broken (likely accept-language / browser system settings, not geolocation). 🟠 Medium

⚠️ Inconvenient

Timecode What was noted
~04:30 On the main page of a logged-in user there is no "Sign in / Register" — there's only "Join Meeting" and "Open App". If you visited again — it's unclear where to click to get into the right flow. After logout, the button appeared: "Now it's Sign in. That's a lot clearer".
~08:00–10:00 EK looked for "change password" in Settings → didn't find it. Went into Help → the AI assistant explained that there are no passwords, login is by one-time code. The Settings UI does not explain this itself — it's empty there. The user logically looks for "change password", doesn't find it, wastes time.

❓ Unclear

Timecode What wasn't understood
~02:30 Where "Sign in" disappears from the main page — at first EK didn't guess that the reason was the already logged-in session. Only realized after logout.
~10:30 After delete + repeated login with the same email: EK is not sure — is this a new account or restoration of the old one. Had to do a second run with notes to make sure the account was actually deleted, not "suspended".

✅ What works well

Registration via email + one-time code

  • Email code arrives "very quick".
  • "My code with your intermind sign-in code expires in 10 minutes or click the button to sign-in directly" — the email wording is clear.
  • The "sign-in directly" button works — no need to enter the code manually.

SSO

  • Google login: "worked perfectly fine on both website and phone" (used before).
  • Google / Microsoft / SSO are clearly shown as options in the UI.

Help / AI assistant

  • When EK didn't understand that there are no passwords, the AI assistant in Help responded quickly and clearly: "Intermind does not use passwords for email logins. You receive a one-time six-digit code…". This saved the scenario.

Delete account — exemplary flow ⭐

EK praises each step in detail: - "Very clear red text", use of trash icon — "universal symbol, very easy to understand". - Clear statement about irreversibility: "very clear statement that it is permanent". - Double confirmation — second pop-up with a warning (important against misclicks). - Email validation on confirmation: when the email is incorrect — "email does not match" (EK checked deliberately). - Big red warning before the final delete: "This will permanently delete your account and all data including meetings, messages, files, team ownership and subscription. This cannot be undone". - Real verification after deletion: EK created data → deleted → logged back in with the same email → "the stuff that I added before isn't there. So that does seem to be working". Deletion actually wipes data.

Logging in again with the same email after delete

  • The account is successfully re-created with the same email — this is correct behavior.
  • What was there before deletion doesn't come back (correct).

📋 Not fully tested

  • Password recovery via "Forgot password" — scenario item §2; EK didn't do it because there are no passwords. The item can be struck from the instructions.
  • Expired / invalid link on the /confirm page — partially checked (clicked the link from the email), but no repeated transition (on a "used link") and no "open after an hour" scenario.
  • Registration via Microsoft SSO — not tried (used Google and email).
  • Inability to register with an already-used email — partially touched on, but without an explicit "tried again and got an error".

🚦 Priorities

  1. 🟠 Language auto-detect in Edge leads to DE locale (00:00). Fix either the detection logic, or show a banner "We detected DE — switch to EN?" instead of a forced redirect.
  2. 🟠 Settings → no "Sign-in method" block (~08:00). Add explicit text "You sign in with one-time codes — passwords are not used" right in settings, don't hide in Help.
  3. 🟡 Main page for a logged-in user — no obvious entry point (~04:30). When the user is already signed in, the main page should not look like a landing.
  4. 🟡 Update tester instructions: strike "password recovery" from §2 scenarios, since there are no passwords in the product — replace with "repeated one-time login".

🎉 What's especially good

Delete account — the best part of Inter-Mind according to EK. This is rare: delete flows are usually designed sloppily. Here it looks carefully worked through — double confirmation, validation, explicit warnings, real data cleanup.
