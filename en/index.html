🧭 Key takeaways

The product is strikingly uneven. In some places you can see careful engineering; in others a feature was shipped to marketing before it was actually built.

What is encouraging

Several parts of the product are done genuinely well:

  • Delete account flow — double confirmation, email validation, real cleanup. EK spent five minutes specifically praising it — in most products this is slapped together.
  • Onboarding — one-time code login, calming music in Pre-Join Setup, Google SSO email consolidation. These are the small things that rarely get attention. Here they got it.
  • Laser pointer on the shared screen where both participants can draw — genuinely nice feature that isn't marketed anywhere.
  • Note editor in chat works cleanly end-to-end (create → share → translate → delete from both sides).

This shows the team has someone who thinks about UX at the level of details.

What is seriously broken

The same product breaks on flagship features:

  • Cloud recording is broken in several places at once: 14 seconds regardless of actual length, the file goes to the participant instead of the creator, the email summary is empty. This is one of the key pricing points — and exactly the one that doesn't work, on several levels.
  • Documents are accessible via direct link without auth — this is a security bug, not UX. After delete it returns raw S3 XML with the key, which only the server should see.
  • Translation — the flagship feature — gives a 5-second lag, sometimes replaces the voice with someone else's, sometimes inserts garbage phrases. The very thing users come for is what suffers most.
  • "Find the UI language switcher" — both testers spent 5 minutes unable to find it. The AI assistant gave instructions that do not lead to success. This is doubly bad: a problem plus a broken lifeline.

Systemic pattern

There's a gap between pricing-page promises and product reality:

  • Domain Management — advertised, missing from UI.
  • Google Drive / OneDrive integration — advertised, missing from UI.
  • AI editor completion — advertised, cannot be found.
  • "17 languages" in docs vs "14 languages" in marketing comparisons — internal contradiction.

The team ships marketing faster than UI. For alpha that's forgivable, but if a paid plan launches tomorrow, this gap turns into chargebacks.

On testing methodology

  • Voice-narrated slicing works. Marker words "doesn't work / inconvenient / unclear" — simple and robust categorization. Testers managed without special training.
  • Cross-tester verification was key. When the same bug is independently seen by two people — it immediately removes "maybe I imagined it" and bumps priority.
  • Whisper hallucinations were the most expensive part of the pipeline. 9 out of 16 files on the first pass were unusable. VAD saved us. Without it we would have spent time on "hallucination reports".
  • Coverage is uneven. §2–4 (registration, profile, meetings) — deep. §10 (recording), §13 (security), §15 (mobile) — shallow: either testers hit a feature they couldn't access (Business+ required) or silently clicked (silent video is useless).

🚦 Recommendations to the team

  1. Urgent — the top-10 critical bugs above (especially recording, document access, deleted meeting reachability).
  2. Sprint — medium bugs (XML error, blank message, language switcher).
  3. Testers — 3 targeted follow-up sessions: §10 (Recording), §13 (Security, Business+ needed), §15 (Mobile, properly).
  4. Tester guide updates — strike out "password recovery" (no passwords); call out that §13–14 require Business+.
