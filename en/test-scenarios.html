Test scenarios: what we check — Inter-Mind

Inter-Mind manual test scenarios

This is a list of user tasks that need to be walked through by hand and recorded on video (see the recording workflow). Each task is what a real user wants to accomplish. It is not a step-by-step instruction: how to perform each task is up to the tester, because exactly «how will a new user figure it out / fail to figure it out» is what we are testing.

The source of the feature list is the page intermind.com/pricing.

0. First contact (landing page)

A real user comes to the site not for the pricing. They want to understand in 30 seconds what this is, and in a minute — try it, before reading any price tags.

  • Open intermind.com for the first time and within 30 seconds understand: what product this is and who it is for.
  • Find on the homepage how to try the product right now, without registering. Count how many clicks to the trial.
  • Find where to ask a question before purchasing (live chat, contacts, FAQ).
  • Read a couple of sections of the landing (what is promised, case studies, testimonials) — does a coherent picture of «why I need this» form.

1. Demo room (try without registering)

The key entry point. On the task «see how well this works for me» the decision «register or not» is made in 1–2 minutes. The product feature list explicitly mentions «Demo mode with auto-cleanup (TTL)» — so this is a real entity.

  • From the homepage find and click the CTA that leads to demo without registration («Try demo», «Try it now», etc.). Measure how many clicks from the homepage to the first spoken line in the demo.
  • Enter the demo room. From the first second understand: I am inside a meeting, translation is on, what is happening.
  • Speak out loud — hear the translation of your own speech or see the transcription.
  • Understand what is available in the demo and what is blocked / labeled «available after registration».
  • Try all the key features in the demo, if they are open: camera, microphone, screen sharing, chat, reactions, recording.
  • Change the translation language on the fly.
  • Exit the demo. See which CTA the product shows on exit (push to registration, to pricing, to the next step).
  • Open the demo result report (/demo/report, if there is a link to it from the exit). Understand what exactly was shown: transcript, translation, any call quality metrics.
  • Return to the same demo room an hour later — is it still alive or has TTL deleted it.
  • Create a second demo room back-to-back — does the product allow this without limits, or does it ask to register.
  • Try the demo from a mobile device (iOS Safari, Android Chrome).

2. Registration and onboarding

  • Register from scratch as a new user (new email).
  • Receive the confirmation email, follow the link to the /confirm page and verify that the confirmation went through; understand where the product takes you next.
  • Open an expired/invalid confirmation link (for example, after following it a second time) — see what the /confirm page shows.
  • Go through the initial onboarding up to the point where it is clear «what to do next».
  • Try to register with an email that is already in use — see what the product responds.
  • Register via SSO / Google (if available).
  • Reset the password via «Forgot password».
  • Sign out and sign back in.
  • Delete your account (if such an action exists in the UI).

3. Profile and team

  • Fill out the profile (name, avatar, interface language).
  • Change the interface language to one of the 17 advertised.
  • Create a team (Team).
  • Invite a single participant to the team by email.
  • Bulk-invite participants via file (txt / csv / json — if such a feature exists).
  • Accept a team invitation from a second account — follow the invitation link (page /invite/[token]): see what is shown to a new user, and what to an already logged-in user.
  • Open an expired / already used / invalid invitation — see what error and what next-step.
  • Change a participant's role (admin / member).
  • Remove a participant from the team.
  • Leave the team yourself.

4. Meetings (core of the product)

Basic video conferencing

  • Create a meeting right now (instant meeting).
  • Schedule a meeting for a future time (if a scheduler exists).
  • Share the meeting link with another person.
  • Join the meeting as the host.
  • Join the meeting as a guest via the link, without an account (if allowed).
  • Join the meeting from a second account in another browser at the same time.
  • Turn the microphone on / off.
  • Turn the camera on / off.
  • Switch the microphone / camera device on the fly.
  • Check FHD video & audio: picture and sound quality in a 1:1 meeting.
  • Leave the meeting.
  • End the meeting as the host (kick everyone out).

Pre-Join Setup

  • Before entering a meeting, land on the Pre-Join Setup screen — preview of camera and microphone. Understand what is shown.
  • On Pre-Join — choose the camera/microphone/speaker device, turn the camera off before joining, turn the microphone off before joining. Join. Verify that the states were preserved.
  • In Settings → Meeting preferences turn off the «Show Pre-Join Dialog» toggle → join the next meeting without Pre-Join. Turn it back on — Pre-Join returns.

Meeting controls and layout

  • Pin a participant — pin someone's video to the main screen.
  • Switch the layout: grid / speaker view (and any others the product offers).
  • Per-participant volume — is there an individual slider.
  • Make another participant a co-host (if such a role exists) and check their permissions.
  • Kick a participant out of the meeting (as host).
  • Force-mute a participant as host.

Basic plan limits

  • Reach the 40-minute limit on a meeting on the Basic plan. See what the product shows before the end, at the moment of the end, and after.

Reactions and interaction

  • Send an emoji reaction during a meeting. Verify that exactly the emojis advertised are available (per the docs — 👍 ❤️ 😂 😮 😢 🙏 + those mentioned in reactions.md). Compare the set with what was promised.
  • «Raise hand» — send hand_raise, see the indicator on another participant, lower it.
  • See reactions and raised hands of other participants.

5. Real-time speech translation

  • In a two-way call (RU ↔︎ EN) turn on translation.
  • Hear the translation of the other party into your language.
  • Change the language on the fly.
  • On Basic — verify that only 1 language pair is available at a time.
  • Try multiple language pairs simultaneously, as far as the number of participants allows (3 people speaking 3 languages is already 3 pairs).
  • Reach the translation hours limit on the plan (Basic — 1 hour). See what the app shows before the end and at the moment of exhaustion.
  • Check the translation of a rare language out of the 17 advertised (not RU, not EN).
  • Check translation of mixed speech within a single phrase (RU + EN interleaved).
  • Check fast speech and whispering — what happens to the translation.
  • Join a meeting where translation has already been running for 5 minutes — will it catch up.

6. Subtitles and transcription

The meeting cockpit has three separate buttons: Translation, Subtitles, Transcription. These are different entities and must be tested separately.

Subtitles

  • Turn on the «Subtitles» button in the meeting cockpit — see live subtitles overlaid on the video.
  • Turn off subtitles — verify that they disappear, while translation/transcription keep working (if they were on).
  • Check subtitles in the second language (if they are sourced from the translation).
  • Check how subtitles overlay on different layouts (grid / speaker view) — whether they cover any important elements.
  • The «on/off» state of subtitles is remembered between meetings — verify.

Transcription

  • During a meeting turn on transcription display on screen — this is a different button, not Subtitles.
  • Read the transcription of your own speech and the other party's speech.
  • Download / save the transcript after the meeting (if such an option exists).
  • Check transcription for a rare language.
  • The «on/off» state of transcription is remembered between meetings — verify.

7. In-meeting chat

  • Write a message in the chat during a meeting.
  • Attach a file to the chat during a meeting.
  • Send an emoji.
  • Copy a link / forward a message.

8. Persistent team chat (outside a meeting)

Messages and history

  • Open the chat history with the team.
  • Write a message in the chat outside a meeting.
  • Receive a notification about a new message.
  • Open a thread a week old — see the history in full.
  • Search across messages.
  • Translate messages (text translation — limit «10K / 200K / 3M words»).

Starting a meeting from the chat

  • In a persistent team chat click «start a meeting» / «call» / a similar button — launch a meeting straight from the conversation.
  • Verify that the chat automatically receives the link to this meeting, and other chat members can join in one click.
  • End the meeting — see what remains in the chat after it ends (recording, transcript, link to summary).
  • Start a meeting from a 1:1 chat and from a group chat — the behavior and UI may differ.
  • Create a meeting from a chat that includes a person from outside the team (external participant) — will they be let in, and what permissions.

Notes in the chat (Note editor)

  • Create a note right from the chat (Note editor — button/command that should be part of the chat).
  • Edit the note, format the text.
  • Share the note with another chat member — see what they see on their side.
  • Close and reopen the note — state and contents preserved.
  • Delete the note.

9. Screen sharing

  • Share the entire screen with another meeting participant.
  • Share a single window.
  • Share a browser tab with audio.
  • Use the laser pointer on the shared screen (if the feature exists).
  • Stop sharing.
  • Share the screen on Safari (there are caveats there — Safari 17+ is required).
  • Try to share from iOS / Android — see what the product shows (per the docs — not available).

10. Recording

Cloud recording

  • Start recording to the cloud during a meeting.
  • Stop recording.
  • Find the ready recording in the personal account / team cloud.
  • Download the recording.
  • Share a link to the recording.
  • Delete the recording.

Local recording

  • Start a local recording (video + tab audio + microphone, as in README).
  • Stop the recording, locate the file.
  • Compare quality: cloud vs local.

Storage limits

  • Understand how much space is used in team storage (1 GB / 30 GB / 1 TB / 10 TB).
  • Upload files up to the limit — see what the product shows.
  • Delete old recordings / files to free up space.

11. Integrations

Reached from the /integrations page. Card visibility depends on role and plan.

Calendar

  • Open Calendar Integration, choose Google Calendar or Outlook Calendar.
  • Create a meeting → click «add to calendar» → verify that the chosen calendar shows an event with the correct time, link, and description.
  • Switch the calendar provider and verify that new creations go to the other calendar.
  • Disconnect the calendar — see what happens to already-created events.

Domain management

  • Add your corporate domain in Domain Management.
  • Copy the DNS TXT and add the record to real DNS (access to the domain registrar is required).
  • Click «Verify» — see the status transition pending → verified.
  • Add a deliberately invalid domain or enter an incorrect TXT — see the invalid status and a clear error.
  • Delete a verified domain.

Cloud storage (Google Drive / OneDrive)

  • Connect Google Drive to the account via OAuth.
  • Upload a file from Inter-Mind to Google Drive (if the feature exists in the UI).
  • Attach a file from Google Drive into a chat / a document.
  • Disconnect Google Drive.
  • Same for OneDrive.

12. Documents, notes, and AI tools

Standalone translator

  • Open /translate — a standalone text translation tool, outside of a conference.
  • Enter a paragraph of text → choose the target language → get the translation.
  • Copy the translation. Share it.
  • Check how the «10K / 200K / 3M words» limit is reflected on this page (if the counter is shared).

Documents (shared documents with access rights)

  • Create a document inside the product.
  • Share the document with a team member — verify that they have access.
  • Share with someone outside the team — see what happens.
  • Revoke access — the user can no longer open it.
  • Open the document via a direct link without authentication — see what is shown.
  • Delete the document.

AI assistant in the editor

  • Inside a document / note invoke AI completion (presumably «continue the text», «summarize», «translate»).
  • Get the model's response — evaluate quality and speed.
  • See what the product shows when the AI does not respond (timeout, error).
  • Is there a limit on AI calls per plan — find it, reach it, see what is said.

13. Security (Business / Enterprise)

  • Configure SSO / SAML for team sign-in.
  • Sign in via corporate SSO.
  • Configure Privacy Zones (whatever this is in the product — find it and understand it).
  • Verify the claim about «Private translation infrastructure» — find where it is described / where it is enabled.

14. Support

  • Open Live chat with support (on all plans) — ask a question, get an answer.
  • On Business / Enterprise — try Live chat & video (video call with support).
  • Find the documentation / help center and resolve a typical question on your own.

15. Mobile experience

  • Open intermind.com on iPhone (Safari). Sign in to the account.
  • Join a meeting from an iPhone, talk to a participant who is on desktop.
  • Turn on translation on mobile.
  • Verify that screen sharing from iOS is not available and the product explains this.
  • Same on Android (Chrome).
  • Go to background / lock the phone during a meeting — see what happens to the audio (per the docs — Safari cuts off background tabs).
  • Catch a network switch wi-fi → 4G → wi-fi during a meeting on mobile.

16. End-to-end «real» scenarios

These are «full run-throughs», as if you were living inside the product. One such run-through is worth more than ten point checks.

  • «Freelancer's first day»: register → create a meeting → invite a colleague → run a 10-minute call with translation → record it → find the recording → sign out.
  • «Small team»: set up a team of 3 → invite all of them → arrange a meeting via chat → run it → share the screen → end it.
  • «Hitting the Basic limit»: on the free plan run a meeting up to the 40-minute limit → see how the product behaves at the cut-off and which upgrade CTA it shows.
  • «Guest without an account»: ask a colleague on another device to join via the link without registering → run a meeting with them.
  • «Long meeting»: a 2-hour meeting with translation and recording — verify that nothing falls off.
  • «Broken internet»: in the middle of a meeting switch wi-fi → 4G → wi-fi. See how long the connection lasts and how it recovers.

What is not in this list

  • Billing and payment (Stripe, upgrade/downgrade, invoices, subscription cancellation) — a separate task that requires a Stripe test environment; we will not run it on live production.
  • Scenarios with large numbers of participants (51+ on Basic, 100+ on Pro, 300+ on Business, 1500 on Enterprise) — impossible to verify with 3 people; this requires either load testing or a group of volunteers.
  • Technical security scenarios (XSS, SQL injection, etc.) — a separate security review task.
  • Point edge cases (broken UTF-8 files, RTL languages, zero-width chars) — we will add them as a separate list if the main scenarios pass cleanly.
